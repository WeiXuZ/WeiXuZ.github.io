总结：

注解

​ 常见注解：

​ @Deprecated 加在方法上，说明该方法已经过时了。显示为一个删除线。

​ @Override 加在方法上，说明方法是重写的父类的方法。如果重写的方法和接口中的方法不一致，说明写错了。帮助我们调错。

​ @SuppressWarnings(“ALL”) 压制警告信息，编辑器在看到这个注解时，忽略警告。

​ 注解有什么作用，怎么用？

​ 注解可以结合反射来使用。可以在类上或方法上设置一些配置信息。可以省去配置文件。

​ 用法：

​ 1、创建一个注解 @interface声明

​ 2、在注解中添加一个属性， 属性写法 类型 属性名()；

​ 3、在需要使用的类上添加注解并指定参数（属性=值）

​ 4、在主方法中，通过反射获取配置的值

​ 创建一个注解：

​ 元注解：

​ 添加在注解声明上的注解。

​ @Retention： 说明这个自定义注解运行在哪个阶段

​ @Target： 说明自定义注解可以添加在什么位置，TYPE表示类上，FIELD表示属性 METHOD表示方法

​ java程序编写、编译、执行时分为三个阶段：

​ 源代码阶段：编写代码 RetentionPolicy.SOURCE

​ 字节码阶段：把源代码编译成字节码文件 RetentionPolicy.CLASS

​ 执行阶段： 把字节码加载进内存进行执行 RetentionPolicy.RUNTIME

​

Servlet

​ 是一个运行在服务器端的Java小程序，可以接受用户发送过来的数据（请求Request），执行Java代码进行处理，最后再给用户返回一些数据（响应Response）。

编写步骤：

​ 1、创建JavaWeb项目

​ 2、编写一个类，实现Servlet接口。

​ 编写一个类，继承GenericServlet抽象类。

​ 编写一个类，继承HttpServlet抽象类。

​ 3、配置路径的映射：

​ 在web.xml中配置：

​ servlet servlet-mapping

​ 使用注解配置 在类上添加@WebServlet(“映射路径”)

Servlet中的生命周期相关方法

​ init： 在Servlet对象创建之后被执行，可以进行初始化操作。只执行1次。

​ 默认情况下，Servlet是在第一次被访问时创建，也可以设置服务器启动时创建

​ load-on-startup的值为正数

​ service：用户每次访问资源时都会执行。

​ destroy：在Servlet即将销毁时执行，可以做资源回收工作。

继承的体系

​ Servlet接口： 有5个需要重写的方法

​ GenericServlet抽象类：有1个需要重写的方法

​ HttpServlet抽象类：根据需要重写doGet或doPost方法

Url-Pattern的配置

​ 1、一个Servlet可以配置多个资源地址

​ 2、配置规则：

  • 精确匹配： /资源名称

  • 目录匹配： /目录名/* 只要目录相同，*表示任意的路径

  • 扩展名匹配： *.扩展名 只要扩展名相同，就可以匹配上

    优先级： 精确匹配 > 目录匹配 > 扩展名匹配

    注意： Servlet的路径要么以/开头, 要么以*开头。

    ​ 目录匹配和扩展名匹配不能同时使用. /*.do

404的解决方案

​ Not Found（请求资源不存在）

​ 1、模块没有发布成功（模块前面是叹号或叉号）

​ 2、模块发布错误（发布的是其他的模块）查看deployment选项卡删除错误的，添加正确的

​ 3、核对模块的映射的虚拟路径错误 设置虚拟路径，在访问时要一致

​ 4、检查是否为Servlet添加了正确的映射地址
